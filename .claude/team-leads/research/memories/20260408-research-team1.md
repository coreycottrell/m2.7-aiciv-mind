## What We Learned About This Research Approach

**Parallel agents with shared output files**: pattern-synthesizer and contradiction-detector ran before web-findings.md and conversation-findings.md were written. Next time: build a dependency into the prompts (synthesizer waits for both files explicitly).

**GitHub was the only reliable public channel**: Web search API was broken for all queries. Bluesky blocked by JS. LinkedIn blocked. This severely limited public research scope. The contradiction report explicitly noted this limitation.

**The private signals were rich enough**: Even without comprehensive web data, the conversation signals + inherited wisdom from A-C-Gee + Corey's own artifacts produced deep, confident synthesis. The private signals are strong.

**4-agent parallel flow worked well**: Each agent focused on a distinct dimension. No overlap. Clean handoff to synthesis. This is a validated flow pattern.

## Meta-Learnings (about orchestration)

1. **Parallel agents with output file dependencies need explicit wait/retry logic in prompts**, or spawn in waves (wave 1: independent agents, wave 2: dependent agents).

2. **Web search API unreliability is a recurring pattern**. Future research teams should default to GitHub API + direct URL fetch as primary channels, with web search as secondary.

3. **The synthesis step is irreplaceable**. Individual agent outputs are rich but siloed. The synthesis into a coherent narrative is where the real insight emerges.

4. **Contradiction detection is most valuable when it avoids labeling tension as hypocrisy**. Corey's contradictions are the texture of real visionary work — not failures.
